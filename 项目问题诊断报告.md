#### 额外 #15 patch 提取用 rsplit 吞掉后续 ``` 围栏 — 已修复
- **问题**：`patch_merger._extract_diff_block` 和 `patch_guard.guard_patch` 用 `rsplit("```", 1)` 提取 diff 块，但 provider 输出里有多个 ` ``` `（` ```diff` 开头、diff 结尾、`## Input Echo` 里 readonly context 的` ```text ` 块）。`rsplit` 从右找最后一个` ``` `，把 Risk Notes、Input Echo 等垃圾内容包进 patch，导致 git apply 失败、guard 误判。
- **修复**：两处都改为 `split("```", 1)[0]`（取 diff 块结尾的第一个` ``` `）。
- **文件**：`src/file_swarm/patch_merger.py`、`src/file_swarm/patch_guard.py`

## 第三轮优化：Patch 规范化 + Prompt 减 Token + 模型 BUG 兼容

### 新增：统一补丁规范化管线 (`patch_normalizer.py`)

将之前分散在各处的 diff 提取、路径修复、hunk 修复逻辑集中到一个模块，并集成到 dispatch 和 merge 管线中。

**处理管线（6 阶段）**：
1. **Markdown 清理** — 找到 `## Patch` 或 ` ```diff ` → 提取 diff 内容 → 在下一个 `##` 标题处截断
2. **围栏净化** — 移除残留的 ` ``` `、` ```diff `、` ```patch ` 行
3. **路径规范化** — 自动补全缺 `a/`/`b/` 前缀的 `---`/`+++` 行
4. **Hunk 行数修复** — 重新计数每个 hunk 的实际 ± 行，修正 LLM 的计数错误（如 GLM-5.1 的 `@@ -6,4 +6,4 @@` → `@@ -6,2 +6,3 @@`）
5. **空白规范化** — 去除前导/尾部空行，统一行尾无空白
6. **结构校验** — 检查是否存在 `---`/`+++`/`@@` 行

**集成点**：
- `dispatcher._run_task`：provider 返回后、guard 之前 → 规范化后再 guard
- `patch_merger.merge_patches`：读取补丁文件后 → 规范化后再合并

### 优化 Prompt 减少主指挥积分损耗

**改前**（~425 tokens header/call）：
```
TASK_ID/TASK_TYPE/USER_REQUEST/GOAL/ASSIGNED_FILES/ALLOWED_FILES (45t)
READ_ONLY_CONTEXT_FILES (full file, unlimited) (variable)
REQUIREMENTS/FORBIDDEN (45t)
HARD_CONSTRAINTS_YAML (full YAML, ~350 chars) (90t)
INTERFACE_CONTRACT_YAML (full YAML, ~200 chars) (50t)
REPO_MAP (~200 chars) (50t)
Instructions (120t)
```

**改后**（~145 tokens header/call）：
```
TASK_ID/TASK/USER_REQUEST/ALLOWED_FILES (50t)
READ_ONLY_CONTEXT (capped at 3000 chars) (variable)
Contracts (condensed rules, ~250 chars) (65t)
Instructions (30t)
```

**节省**：约 280 tokens/call × 6 workers = **~1680 tokens/轮**（约 65% boilerplate 减少）。

### 修复过程中暴露的自引入 Bug

| Bug | 原因 | 修复 |
|-----|------|------|
| markdown stripper 误吞 diff 内容 | `## Patch` 被识别为"终止标记"而 break | 改为 `## Patch` 跳过继续、其它 `##` 才 break |
| hunk 尾部空行被误算为上下文 | merge 的 `\n` 分隔符被计入 hunk body | hunk 计数前先 `pop()` 尾部空行 |
| 多处重复添加 `\n` 导致多空行 | 每个子函数独立加 `\n` | 统一只在 `normalize_patch` 顶层加一次 |
| `_extract_context_file` 格式不兼容 | 新 prompt 用 `READ_ONLY_CONTEXT:` 而非 `READ_ONLY_CONTEXT_FILES:` | 兼容两种格式 |
